# 0.6.1 — 2026-06-02

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.16 → ^0.9.21` — picks up three framework fixes:
  - Per-request logs and traces now carry fresh request + trace/span IDs instead of the frozen boot context (HTTP transport log context fix).
  - `fetchWithTimeout` strips query-string secrets (e.g. `?api_key=`) from error messages and logs before surfacing them.
  - `withRetry` fails fast on non-retryable errors; `ctx.fail` auto-populates the `retryable` flag.
- **README client-config key** renamed from `nws-weather` to `nws-weather-mcp-server` in all four install examples (hosted HTTP, bunx stdio, npx stdio, Docker stdio).
- **Skills synced** to `@cyanheads/mcp-ts-core` 0.9.21: `add-tool`, `add-service`, `api-canvas`, `api-context`, `api-linter`, `api-utils`, `design-mcp-server`, `release-and-publish` updated; `api-mirror` and `orchestrations` added.
- **`scripts/devcheck.ts`** updated to 0.9.21 framework conventions; `scripts/check-skill-versions.ts` and `scripts/release-github.ts` added.
