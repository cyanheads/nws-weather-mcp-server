# 0.6.2 — 2026-06-12

## Added

- **`scripts/clean-mcpb.ts`** — post-pack MCPB cleaner wired into the `bundle` script. Runs `mcpb clean` then strips dependency-shipped agent-doc entries (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) nested under `node_modules/`, which root-anchored `.mcpbignore` patterns cannot reach.
- **`lint-packaging.ts` checks 5–9** — bundle-content guards (root dev dirs excluded after `.mcpbignore` evaluation, no unanchored patterns that also strip runtime `node_modules/**` paths, no stripping of critical runtime package paths, zero agent-doc entries in a built `.mcpb`) plus an identity check that `name`/`title` in `createApp()` and manifest `display_name` equal the unscoped package name.
- **`check-framework-antipatterns.ts` rule 4** — flags `z.coerce.boolean()` on env flags (`Boolean("false") === true`, so the variable can't be disabled via env; use `z.stringbool()`); comment lines are skipped so JSDoc naming the pattern doesn't false-positive.
- **Docker `HEALTHCHECK`** — polls `/healthz` every 30s via `bun -e fetch` (no curl/wget); `ARG APP_VERSION` feeds the `org.opencontainers.image.version` OCI label.
- **`techniques` skill** — catalog of reusable response/data-shaping patterns (outline-on-overflow), synced from the framework.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21 → ^0.10.6`.
- **Server identity** — `createApp()` now sets `name` and `title` explicitly to `nws-weather-mcp-server`; `.codex-plugin` `interface.displayName` renamed `@cyanheads/nws-weather-mcp-server → nws-weather-mcp-server` so the hyphenated repo name is the display identity on every surface.
- **`.mcpbignore`** — root dev-dir patterns anchored (`/.claude/`, `/.agents/`, `/skills/`) so they exclude only the project's own trees, not runtime paths shipped inside dependencies.
- **Skills synced** to `@cyanheads/mcp-ts-core` 0.10.6 across `skills/`, `.claude/skills/`, `.agents/skills/`; `devcheck.config.json` adds `skillVersions.ignore` for seven skills whose bodies were re-synced without a framework `metadata.version` bump.
