# 0.5.10 — 2026-05-22

Maintenance release. Picks up `@cyanheads/mcp-ts-core` through 0.9.4, adds `zod` as an explicit dependency (required by the 0.9.2 peerDep promotion), and applies doc/meta conventions from the latest `polish-docs-meta` and `maintenance` skill passes.

## Added

- **`zod` explicit dependency** `^4.4.3` — required by the `@cyanheads/mcp-ts-core` 0.9.2 peerDep promotion ([cyanheads/mcp-ts-core#143](https://github.com/cyanheads/mcp-ts-core/issues/143)). Was previously resolved transitively; now declared directly in `dependencies`.
- **`MCP_GC_PRESSURE_INTERVAL_MS`** documented in `.env.example` — opt-in Bun-only forced-GC loop ([cyanheads/mcp-ts-core#50](https://github.com/cyanheads/mcp-ts-core/issues/50)) for HTTP deployments exhibiting heap growth. Default `0` (disabled); recommended starting point: `60000`.
- **README install buttons** — Cursor `https://cursor.com/en/install-mcp` deep-link badge + VS Code `https://vscode.dev/redirect?url=vscode:mcp/install?...` badge. Framework spotlight badge recolored cyan-300 (`67E8F9`), promoted to its own row.
- **New scripts**: `audit:refresh` (lockfile refresh + re-audit), `list-skills` (skill index), `lint:packaging` (env-var alignment check), `publish-mcp` (MCP Registry publish via Keychain PAT).

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.1 → ^0.9.4`**. Notable upstream changes across 0.9.2–0.9.4: `zod` peerDep promotion; MCPB bundle support (`lint:packaging`); `multi-server-orchestration` skill + `list-skills`; `field-test` zsh `status` clash fix + concurrent-session isolation; `format-parity` lint walks all union branches; `MCP_GC_PRESSURE_INTERVAL_MS` GC loop; skill-versioning policy covers reference files.
- **README badge layout restructured** — version/license/docker/MCP SDK/npm/TS/Bun badges on one row; install buttons on a second row; Framework spotlight on a third. Surface count separator `·` → `•`.
- **Action-first descriptions** — `package.json` `description` and `server.json` `description` switch to verb-led phrasing per `polish-docs-meta` 2.0 conventions.
- **Skills synced from framework 0.9.2–0.9.4** — `maintenance` 2.3, `field-test` 2.5, `polish-docs-meta` 2.0 (+ references), `release-and-publish` 2.4. Updated into both `skills/` and `.claude/skills/`.
- **CLAUDE.md / AGENTS.md** — Added `audit:refresh` command entry; updated skill table descriptions to match synced skill versions.
- **`@types/node` `^25.8.0 → ^25.9.1`**, **`@vitest/coverage-istanbul` `^4.1.6 → ^4.1.7`**, **`vitest` `^4.1.6 → ^4.1.7`**.
