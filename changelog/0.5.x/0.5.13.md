# 0.5.13 — 2026-05-30

## Added

- **`enrichment` block on `nws_search_alerts`** — `totalCount`, `shownCount`, `appliedFilters`, and optional `notice` now surface via `ctx.enrich()`, reaching both `structuredContent` and `content[]`. Empty-result `notice` echoes applied filters and recovery hints.
- **`enrichment` block on `nws_find_stations`** — `totalFound` (pre-limit station count), `totalCount` (returned), and optional `notice` on empty results.
- **`nws_find_stations` `totalFound`** — `NwsService.findStations` now returns the true count of nearby stations before the `limit` is applied, distinct from the returned count.

## Changed

- **`nws_search_alerts` `structuredContent`** — `count`, `shown`, `filters` removed from the domain `output` schema; their values now surface under `enrichment` as `totalCount`, `shownCount`, `appliedFilters`.
- **`@cyanheads/mcp-ts-core` `^0.9.13 → ^0.9.16`** — picks up enrichment contract support (`enrichment` + `enrichmentTrailer` builder fields, `ctx.enrich` helpers).
