# 0.7.1 ‚Äî 2026-06-30

Framework-maintenance release ‚Äî adopts `@cyanheads/mcp-ts-core` `^0.10.10` (a security-driven transitive-dependency refresh) and tightens two tool descriptions per [#22](https://github.com/cyanheads/nws-weather-mcp-server/issues/22). No tool schema, handler, or runtime behavior changed.

## Changed

- **`nws_get_forecast` description** ‚Äî dropped the internal routing note "Internally resolves coordinates to the NWS grid" from the tool description and "Truncated to 4 decimal places" from the `latitude`/`longitude` field descriptions. The tool-selection contract (coordinate coverage, `hourly` vs. named 12-hour periods) is unchanged; the routing detail lives in code comments and `docs/design.md`. ([#22](https://github.com/cyanheads/nws-weather-mcp-server/issues/22))
- **`nws_get_office_discussion` description** ‚Äî dropped the two-hop products-API note ("Fetches the two-hop products API: list endpoint first‚Ä¶"). Product types (AFD/HWO/ZFP/SPS) and the `office`-code chaining hint from `nws_get_forecast` are unchanged. ([#22](https://github.com/cyanheads/nws-weather-mcp-server/issues/22))

## Security

The `^0.10.10` bump and lockfile re-resolve clear 8 transitive advisories (2 high, 6 moderate); `bun audit` now reports 0 (was 8). All were transitive ‚Äî no direct dependency was affected.

- **`hono` `<4.12.25` ‚Üí `4.12.27`** (via `@cyanheads/mcp-ts-core`) ‚Äî CORS middleware reflects any Origin with credentials on the wildcard default (**high**); `serve-static` path traversal on Windows via encoded backslash; AWS Lambda adapter merges `Set-Cookie` headers and drops cookies; Body Limit bypass on AWS Lambda via understated `Content-Length`; Lambda@Edge adapter drops repeated request headers.
- **`vite` `<=8.0.15` ‚Üí `8.1.2`** (via `vitest`) ‚Äî `server.fs.deny` bypass on Windows alternate paths (**high**); `launch-editor` NTLMv2 hash disclosure via UNC path handling on Windows.
- **`js-yaml` `<3.15.0` ‚Üí `3.15.0`** (via the `depcheck` chain) ‚Äî quadratic-complexity DoS in merge-key handling via repeated aliases. `mcp-ts-core` `0.10.10` also moved `js-yaml` to an optional peer, so the framework's `js-yaml` chain drops out of this server's tree entirely.
