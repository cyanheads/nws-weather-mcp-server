# 0.6.0 ‚Äî 2026-05-30

## Added

- **`nws_get_office_discussion`** ‚Äî fetches the latest narrative product (AFD, HWO, ZFP, or SPS) from a Weather Forecast Office via `GET /products/types/{type}/locations/{wfo}`, returning product text, issuance time, and WFO metadata. ([#9](https://github.com/cyanheads/nws-weather-mcp-server/issues/9))
- **`nws_get_zone_forecast`** ‚Äî fetches zone-level text forecast periods by public NWS forecast-zone code via `GET /zones/{type}/{zoneId}/forecast`, completing the alert‚Üíforecast chain (alerts return zone codes; this tool resolves them). ([#9](https://github.com/cyanheads/nws-weather-mcp-server/issues/9))
- **`nws_get_forecast` enrichment** ‚Äî `periodCount`, mode echo (`hourly` vs `7-day`), and `generatedAt` so agents know forecast granularity and currency without re-deriving from the array. ([#15](https://github.com/cyanheads/nws-weather-mcp-server/issues/15))
- **`nws_get_observations` enrichment** ‚Äî station echo and a `ctx.enrich.notice()` when the latest observation is stale, flagging low-confidence current conditions. ([#15](https://github.com/cyanheads/nws-weather-mcp-server/issues/15))

## Changed

- **`MAX_RETRIES`** now applied to the coordinate-path station-list fetch in `getObservation()`, `findStations()`, and `listAlertTypes()` ‚Äî transient NWS 500s on these endpoints now recover silently instead of surfacing `ServiceUnavailable`. ([#13](https://github.com/cyanheads/nws-weather-mcp-server/issues/13))

## Fixed

- **`nws_get_observations` direct-station path** ‚Äî a 404 from `observations/latest` now surfaces `reason: no_observations` (station is valid but has no current data) instead of `station_not_found`, preventing agents from looping to re-discover a valid station ID. ([#11](https://github.com/cyanheads/nws-weather-mcp-server/issues/11), [#12](https://github.com/cyanheads/nws-weather-mcp-server/issues/12))
