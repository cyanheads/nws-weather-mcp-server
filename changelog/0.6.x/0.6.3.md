# 0.6.3 — 2026-06-15

## Fixed

- **`nws_get_observations`** — `structuredContent` numeric fields (`temperatureC`, `relativeHumidityPct`, `cloudLayers[].baseM`, etc.) now round via `Math.round` to match `format()`, so both client surfaces report identical values instead of the structured channel leaking raw upstream floats. An empty `textDescription` (common during METAR gaps) no longer renders a bare `****` in the header. ([#16](https://github.com/cyanheads/nws-weather-mcp-server/issues/16))
- **`nws_get_office_discussion`** — an empty product list (`{"@graph":[]}`) is ambiguous: both an unknown office code and a valid office with no current product return HTTP 200 with an empty graph. The handler now probes `/offices/{officeId}` to branch the error — a valid office reports that episodic products (SPS, HWO) are commonly empty and points to `AFD`, while an unknown code still prompts to verify the WFO code. ([#17](https://github.com/cyanheads/nws-weather-mcp-server/issues/17))
