# 0.5.12 — 2026-05-28

Maintenance release adopting `@cyanheads/mcp-ts-core` through 0.9.13 with targeted server-specific changes.

## Added

- **`MCP_HTTP_MAX_BODY_BYTES`** — configurable inbound request-body cap on the HTTP MCP endpoint. Oversized requests are rejected with `413` before the body is parsed. Default 1 MiB; set to `0` to defer to the runtime/reverse proxy. Documented in `.env.example`.
- **`package.json` keywords** — `bun`, `stdio`, `streamable-http` added. These now surface on the `GET /mcp` status endpoint and are searchable via `package.json` metadata consumers.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.6 → ^0.9.13`** — picks up HTTP transport hardening (session-init gate returns 400 for stateful sessions missing `Mcp-Session-Id`; 401/403/400/404 log at `warning` instead of routing through the full error pipeline with stack traces; `GET /mcp` surfaces `package.json` keywords).
- **`landing.requireAuth: false`** — set explicitly in `createApp()` so the tool/resource inventory is served to unauthenticated callers. Required now that `landing.requireAuth` defaults to `true` when `MCP_AUTH_MODE` is `jwt` or `oauth`.
- **Skills synced** — `code-simplifier`, `git-wrapup`, `release-and-publish`, `api-linter` added; `migrate-mcp-ts-template` removed; `maintenance`, `polish-docs-meta`, `api-config` updated.
- **Agent protocol** — plugin metadata checklist items and "close the loop on issues" rule added to `CLAUDE.md` and `AGENTS.md`.
- **`@biomejs/biome`** `^2.4.15 → ^2.4.16`.
